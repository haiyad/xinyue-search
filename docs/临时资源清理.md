## 临时资源清理机制
### 1. 清理方法位置
清理功能位于 app/api/controller/Other.php 的 delete_search() 方法
### 2. 清理规则
- 清理条件 ： is_time = 1 （临时资源）且 update_time <= time() - (30 * 60) （30分钟前更新）
- 清理内容 ：
  - 删除数据库中的资源记录
  - 调用对应网盘的API删除网盘中的实际文件
- 处理方式 ：使用 chunk() 方法分批处理，每批100条，避免内存溢出
### 3. 触发方式
重要 ：项目本身 没有内置的定时任务机制 ，需要通过外部方式定时触发：
宝塔面板计划任务
1. 登录宝塔面板
2. 进入"计划任务"
3. 添加任务类型为"访问URL"
4. URL地址： http://yourdomain.com/api/other/delete_search
5. 执行周期：每30分钟
### 4. 临时资源标识
在数据库中，临时资源通过以下字段标识：

- is_time ：0=正常资源，1=临时资源
- update_time ：最后访问/更新时间，用于判断是否过期
