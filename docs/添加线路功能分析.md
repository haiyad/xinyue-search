# 添加线路功能分析

## 一、功能概述

添加线路功能用于配置搜索资源的数据来源，支持三种类型的线路：
- **API接口**：调用第三方API获取资源
- **TG频道**：从Telegram频道爬取资源
- **网页爬虫**：从网页爬取资源

## 二、添加线路完整流程

### 2.1 前端流程

```
用户点击"添加"按钮
    ↓
clickAdd() 方法被触发
    ↓
初始化表单默认值
    ↓
打开添加对话框 (dialogFormAdd = true)
    ↓
用户填写表单
    ↓
用户点击"确认添加"
    ↓
postAdd() 方法被触发
    ↓
表单验证 (this.$refs['formAdd'].validate)
    ↓
发送POST请求到 /admin/api_list/add
    ↓
后端处理并返回结果
    ↓
前端显示成功/失败提示
    ↓
关闭对话框并刷新列表
```

### 2.2 后端流程

```
接收POST请求
    ↓
access() 权限校验
    ↓
validateInsertFields() 必填字段校验
    ↓
getInsertDataFromRequest() 提取允许的字段
    ↓
添加时间戳 (create_time, update_time)
    ↓
插入数据库 (model->insertGetId)
    ↓
返回成功响应
```

## 四、线路类型详解

### 4.1 API接口类型

**适用场景**: 对接第三方搜索API

**配置字段**:
- `pantype`: 网盘类型（0/1/2/3/4）
- `name`: 线路名称
- `type`: "api"
- `url`: 接口地址
- `method`: "GET" 或 "POST"
- `headers`: 请求头（JSON格式）
- `fixed_params`: 接口参数（JSON格式）
  ```json
  {
    "search": "{keyword}"
  }
  ```
- `field_map`: 字段映射（JSON格式）
  ```json
  {
    "list_path": "data.list",
    "fields": {
      "title": "资源名称",
      "url": "资源地址"
    }
  }
  ```
- `count`: 获取资源数
- `weight`: 排序
- `status`: 启用状态

**处理流程**:
```
接收搜索关键词
    ↓
替换 {keyword} 占位符
    ↓
发送HTTP请求到接口地址
    ↓
解析JSON响应
    ↓
根据 field_map 提取字段
    ↓
使用正则表达式提取网盘链接
    ↓
返回结果列表
```

### 4.2 TG频道类型

**适用场景**: 从Telegram频道获取资源

**配置字段**:
- `pantype`: 网盘类型
- `name`: 线路名称
- `type`: "tg"
- `url`: TG频道名称（如 "NewQuark"）
- `count`: 获取资源数
- `weight`: 排序
- `status`: 启用状态

**处理流程**:
```
构建TG URL: https://t.me/s/{频道名}?q={搜索词}
    ↓
获取HTML内容
    ↓
使用XPath提取消息内容
    ↓
从消息中提取标题和网盘链接
    ↓
使用正则表达式匹配网盘链接
    ↓
返回结果列表
```

**限制**: 国内服务器无法使用

### 4.3 网页爬虫类型

**适用场景**: 从网页爬取资源

**配置字段**:
- `pantype`: 网盘类型
- `name`: 线路名称
- `type`: "html"
- `url`: 目标网址（支持{keyword}）
  ```
  https://www.example.com/search?q={keyword}
  ```
- `html_item`: 内容标签（格式：标签名+class名）
  ```
  div+item
  ```
- `html_title`: 标题标签（格式：标签名+class名）
  ```
  h3+title
  ```
- `html_type`: 是否需要详情页（0不需要 1需要）
- `html_url`: 详情页标签（格式：a+class名）
  ```
  a+link
  ```
- `html_url2`: 网盘链接标签（格式：标签名+class名）
  ```
  a+url
  ```
- `count`: 获取资源数
- `weight`: 排序
- `status`: 启用状态

**处理流程**:
```
替换 {keyword} 占位符
    ↓
获取网页HTML
    ↓
使用XPath提取列表项
    ↓
提取标题和网盘链接
    ↓
如果需要详情页：
    ├─ 提取详情页URL
    ├─ 访问详情页
    └─ 从详情页提取网盘链接
    ↓
使用正则表达式匹配网盘链接
    ↓
返回结果列表
```

## 五、线路使用流程

### 5.1 搜索时使用线路

```
用户在前台搜索
    ↓
调用 /api/other/web_search
    ↓
传递参数: title, is_type, is_show
    ↓
后端查询启用的线路
    ↓
按权重降序排序
    ↓
遍历每个线路：
    ├─ API类型 → handleApi()
    ├─ TG类型 → handleTg()
    └─ 网页类型 → handleWeb()
    ↓
提取网盘链接和标题
    ↓
判断是否加密URL
    ↓
通过SSE实时推送结果
    ↓
前端展示资源列表
```

### 5.2 线路查询逻辑

```php
// 查询启用的线路
$lines = $this->ApiListModel
    ->where('status', 1)
    ->where('pantype', $is_type)
    ->order('weight desc')
    ->select()
    ->toArray();

// 合并自定义线路
$lines = array_merge($this->getCustomLines(), $lines);
```

**查询条件**:
- `status = 1`: 线路必须启用
- `pantype = $is_type`: 匹配用户选择的网盘类型
- `order by weight desc`: 按权重降序排序

## 六、增加新网盘类型线路的步骤

### 6.1 前提条件

1. 网盘类型已在 `config/pan_types.php` 中配置
2. 网盘处理类已创建（在 `extend/netdisk/pan/` 目录）
3. Cookie配置已添加到数据库

### 6.2 添加API类型线路

**步骤1**: 进入接口配置页面
- 路径: `/qfadmin/source/apilist`
- 点击"添加"按钮

**步骤2**: 填写表单
```
网盘类型: 选择新网盘类型（如天翼云盘）
线路名称: 输入线路名称（如"天翼搜索线路"）
类型: 选择"API接口"
总数限制: 设置获取资源数（建议5）
接口地址: 输入API地址
请求方式: 选择GET或POST
请求头: 输入JSON格式的请求头
接口参数: 输入JSON格式的参数
  {
    "search": "{keyword}"
  }
字段映射: 输入JSON格式的字段映射
  {
    "list_path": "data.list",
    "fields": {
      "title": "资源名称",
      "url": "资源地址"
    }
  }
排序: 设置权重（数字越大越靠前）
是否启用: 开启或关闭
```

**步骤3**: 点击"确认添加"
- 系统会验证必填字段
- 插入数据库
- 刷新列表

### 6.3 添加TG频道类型线路

**步骤1**: 填写表单
```
网盘类型: 选择新网盘类型
线路名称: 输入线路名称
类型: 选择"TG频道"
总数限制: 设置获取资源数
TG频道: 输入频道名称（如"NewQuark"）
排序: 设置权重
是否启用: 开启或关闭
```

**步骤2**: 点击"确认添加"

### 6.4 添加网页爬虫类型线路

**步骤1**: 填写表单
```
网盘类型: 选择新网盘类型
线路名称: 输入线路名称
类型: 选择"网页爬虫"
总数限制: 设置获取资源数
目标网址: 输入网址（支持{keyword}）
  https://www.example.com/search?q={keyword}
内容标签: 输入标签（如"div+item"）
标题标签: 输入标签（如"h3+title"）
详情页: 选择是否需要进入详情页
详情页标签: 如果需要详情页，输入标签
网盘链接: 输入标签（如"a+url"）
排序: 设置权重
是否启用: 开启或关闭
```

**步骤2**: 点击"确认添加"

## 九、常见问题

### 9.1 线路不生效

**可能原因**:
1. 线路状态未启用
2. 网盘类型不匹配
3. 权重设置过低
4. 接口地址错误
5. 字段映射配置错误

**解决方法**:
1. 检查线路状态是否为启用
2. 确认网盘类型选择正确
3. 调整权重值
4. 验证接口地址可访问
5. 检查字段映射是否正确

### 9.2 获取不到数据

**可能原因**:
1. 接口返回格式错误
2. 字段映射路径错误
3. 网盘链接正则不匹配
4. Cookie配置错误
5. 网络问题

**解决方法**:
1. 查看接口返回的实际格式
2. 调整字段映射路径
3. 检查URL正则表达式
4. 验证Cookie配置
5. 检查网络连接

### 9.3 URL提取失败

**可能原因**:
1. URL正则表达式不正确
2. 网盘链接格式变化
3. API返回的URL不完整

**解决方法**:
1. 更新URL正则表达式
2. 检查网盘链接格式
3. 调试API返回数据

## 十、最佳实践

### 10.1 线路配置

1. **合理设置权重**: 重要线路设置较高权重
2. **限制获取数量**: 建议设置为5，避免过多请求
3. **使用HTTPS**: 接口地址使用HTTPS协议
4. **设置超时**: 接口响应时间不宜过长
5. **错误处理**: 配置备用线路

### 10.2 测试验证

1. **先测试后添加**: 确保接口可用
2. **多种测试词**: 测试不同关键词
3. **监控日志**: 观察线路运行情况
4. **定期检查**: 定期验证线路有效性

### 10.3 维护管理

1. **及时更新**: 接口变化时及时更新配置
2. **禁用失效线路**: 发现问题及时禁用
3. **备份配置**: 重要配置做好备份
4. **文档记录**: 记录线路配置说明

## 十一、总结

添加线路功能提供了灵活的数据源配置方式，支持多种类型的线路。增加新网盘类型线路时：

1. **确保网盘类型已配置**: 在 `config/pan_types.php` 中
2. **选择合适的线路类型**: API/TG/网页
3. **正确填写配置**: 根据线路类型填写对应字段
4. **测试验证**: 添加后进行测试
5. **监控维护**: 定期检查线路状态

通过合理配置线路，可以丰富搜索资源来源，提高搜索效果。
